Git Bash
  • whoami = gives user
  • pwd = print working directory
  • git config —global user.name “Username_here”
  • git config —global user.email “email_here”
  • git config —list
  • git config —global alias.logb “log —oneline —graph —all” = creates a (shortened) alias for a command
  • git status = check tracked and untracked file

    git status —short = used to get status of files in short

  • git add file_name
    • git add —all = adds all files in working directory
    • git add -A = shorter version
    • better to add as git add file_name1 file_name2
  • git restore file_name - restore modified file
  • git commit -m “message_here”

    git commit -a -m “message_here” = commits all files without needing to stage it (add)

  • git command -help = help for command
  • git help —all = help on all
  • git branch branch_name = Create a branch with that name

    git branch = show all branches also tells current branch

    git checkout branch_name = go to this branch

    git branch -d branch_name = delete branch

  • git log = shows a log of all commits

    git log origin/master = all commits on master

    git log —oneline = shows log of all commits with shortened hashs (#)

    git log —online —graph —all = shows all branches

  • For merging

    git checkout master = be at master

    git merge branch_name = branch merged to master

  • git reset = removes staged changes same as when used with -mixed

    git reset # -soft = resets to the specified ‘#’ from log, but does not change files, ie only moves the head back to selected ‘#’

    git reset # -hard = resets to the specified ‘#’ from log, changes all files to chosen ‘#’ state

  • git revert = creates a copy of history on head as commit

    git revert # —no-edit = creates a copy to chosen ‘#’ at HEAD.

    git add * = stage the changes

    git commit —no-edit = create the revert

    useful when multiple users are working on a commit as it does not affect their task.

  • git remote = shows all remote repositories

    git remote -v = location of where it is pointing at

    git remote add origin link.git = link new github repo to existing open repo locally

  • git push —set-upstream origin master = push new repo
  • to pull
    • git fetch origin = update chages on origin
    • git merge origin/master = merge to local

    git pull origin = pulls all changes from origin

  • git push origin = push changes
  • git branch -a = after pulling to use remote branches after checkout to it
  • git checkout -b branch_name = create and switch to this branch
  • git push origin branch_name = push new branch
  • git reset hash = replace hash with the hash to reset to that commit
  • .gitignore

    uses # for commenting

    *.txt to ignore all .txt files

    temp/ = ignore all files in dir temp

  • git clone link = clone the repo in link
  • git fetch = get all change references (not files itself- merge for getting files) from remote repo
  • git push -u origin branch_name = -u keeps local reference and origin is where it is pushed = push changes from a different local repo to remote
Markdown
  • biggest heading → #
  • mid heading → ##
  • small heading → ###
  • bold → ** or __(2 under) and text in between
  • italics →* or _(1 under) and text in between
  • bold italics → ***
  • strikethrough → ~~ and text in between
  • <sub>…</sub> → subscript contained text
  • <sup>…</sup> → superscript contained text
  • to quote → use > for para
  • nested block quote → use >> after > to contain
  • start with 1. → ordered list
  • start with + or - or * → unordered list
  • on empty line *** or ___(3 under) for horizontal rule
  • leave line above and below code to make it code block
  • to add link → [link_text](link_site “optional title, seen when hovered”)
  • <link_site> → to direct add full link
  • for img → [alt_text](img_url)
